# 1차: 대화형 요약 (5,000자 이내) - Chunk 처리로 길이 초과 오류 해결
def first_summary(contents):
    # 길이 초과 시 Chunk로 분할 처리
    if len(contents) > 5000:
        chunks = split_text_into_chunks(contents, 5000)
        summary_parts = []
        for chunk in chunks:
            prompt = chunk + "\n\n 위 내용을 대화형으로 요약해주세요."
            completion = openai.ChatCompletion.create(
                model="gpt-3.5-turbo-16k-0613",
                messages=[{"role": "user", "content": prompt}]
            )
            summary_parts.append(completion['choices'][0]['message']['content'].strip())
        return " ".join(summary_parts)
    else:
        prompt = contents + "\n\n 위 내용을 대화형으로 요약해주세요."
        completion = openai.ChatCompletion.create(
            model="gpt-3.5-turbo-16k-0613",
            messages=[{"role": "user", "content": prompt}]
        )
        return completion['choices'][0]['message']['content'].strip()

# Chunk 분할 함수
def split_text_into_chunks(text, chunk_size):
    chunks = []
    for i in range(0, len(text), chunk_size):
        chunks.append(text[i:i+chunk_size])
    return chunks

  • 상담사 별 주간, 월간 누적 콜수 집계
  • STT 내역 조회

4️⃣ VOC 자동 평가

많은 VOC를 각각 분석하기에 인력 리소스가 부족한 상황

  • 8가지 평가 기준 자동화 (기본인사, 문의파악, ID확인, 종료인사 등)
  • 고객반응도 자동 분류 (불만/만족/보통)

유선상담 녹취를 STT로 추출한 결과값을 적극 활용방안 모색

  • GPT 일 평균 사용비용 25$ → 5$ (80% 감소)
  • 모든 내역에 대한 추출 Format 일치율 분석 98% 이상
  • 상담사 / 고객 분류에 따른 내용 요약 5% 이하

6️⃣ VIP 상담내역 CRM기입

현재 VIP고객 데이터의 유실이 많은 상황으로 기록이 가능한 요소들은 전부 CRM활용 목적의 자산화

  • STT로 분석된 상담내용을 바탕으로 VIP고객의 주요 CS를 파악
  • 불만키워드 선감지 및 방문미팅시 주요 아웃다 도출

# MySQL에서 상담 데이터 조회
def fetch_consultation_data():
    conn = pymysql.connect(host='internal-db-server.company.com', 
                          user='stt_user', 
                          password='stt_password_2023', 
                          db='customer_service_db')
    cursor = conn.cursor()
    
    # sa_cti_summary 테이블에서 상담 데이터 조회
    select_query = """SELECT Collect_Date, manager, contents, summary, filename, mall_id 
                                   FROM sa_cti_summary 
                                   WHERE chk_summary = 'F' 
                                   ORDER BY Collect_Date DESC"""
    cursor.execute(select_query)
    results = cursor.fetchall()
    
    # JSON 데이터 생성
    json_items = []
    for row in results:
        json_item = {
            "collect_date": row[0].strftime('%Y-%m-%d') if row[0] else None,
            "manager": row[1],
            "contents": row[2],
            "summary": row[3],
            "filename": row[4],
            "mall_id": row[5]
        }
        json_items.append(json_item)
    
    # JSON 파일로 저장
    output_path = "consultation_data.json"
    with open(output_path, 'w', encoding='utf-8') as json_file:
        json.dump(json_items, json_file, ensure_ascii=False, indent=4)
    
    print(f"총 {len(json_items)}건의 상담 데이터를 조회했습니다.")
    return output_path, len(json_items)

# Vector Store에 파일 업로드 및 관리
def upload_to_vector_store(file_path):
    client = openai.OpenAI(api_key=OPENAI_API_KEY)
    
    # Vector Store ID (실제 환경에서는 환경변수로 관리)
    vector_store_id = "vs_consultation_data_2023"
    
    # 기존 파일들 삭제
    vector_store_files = client.beta.vector_stores.files.list(
        vector_store_id=vector_store_id
    )
    
    print(f"기존 파일 {len(vector_store_files.data)}개 삭제 중...")
    for file in vector_store_files.data:
        client.beta.vector_stores.files.delete(
            vector_store_id=vector_store_id,
            file_id=file.id
        )
    
    # 새 파일 업로드
    print("상담 데이터를 Vector Store에 업로드 중...")
    with open(file_path, "rb") as file_stream:
        file_batch = client.beta.vector_stores.file_batches.upload_and_poll(
            vector_store_id=vector_store_id, 
            files=[file_stream]
        )
    
    # 업로드 상태 확인
    if file_batch.status == 'failed':
        for file in file_batch.files:
            if file.status == 'failed':
                print(f"File ID: {file.id}, Reason: {file.failure_reason}")
        raise Exception("Vector Store 파일 업로드 실패")
    
    print("Vector Store 업로드가 완료되었습니다.")
    return file_batch
